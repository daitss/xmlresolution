An Example PREMIS XML Document

When an XML document is uploaded for a particular collection the service responds with a PREMIS XML document summarizing the analysis of the document. The returned XML document should validate correctly, though it is expected only certain parts of it will be of value to the client, which commonly extracts parts of the document. The elements object, event, and agents are included as immediate children of the root node.

    </objectIdentifier>
    <objectCharacteristics>
      <compositionLevel>0</compositionLevel>
      <fixity>
        <messageDigestAlgorithm>MD5</messageDigestAlgorithm>
        <messageDigest>df6912619ff3041a8d6f4a5e9964b6f2</messageDigest>
      </fixity>

      <format>
        <formatDesignation>
          <formatName>XML</formatName>
          <formatVersion>1.0</formatVersion>

          <formatRegistryName>http://www.nationalarchives.gov.uk/pronom</formatRegistryName>
          <formatRegistryKey>fmt/101</formatRegistryKey>

    </objectCharacteristics>
    <linkingEventIdentifier>

</premis>
