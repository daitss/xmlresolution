An Example Manifest XML Document

A manifest XML document is constructed and returned in the tar file of schemas associated with the collection. There is no schema associated with the manifest document. The root element <resolutions> act as a container for individual <resolutions> - these latter are associated with exactly one XML document, and list information about the schemas used to resolve that document. The status associated with the schemas are one of success, redirect, unresolved, and failure. These indicate, respectively, that the attempt to acquire the schema file resulted in a successful download; resulted in a redirect to another location; that there was not enough location information to retrieve a schema; or lastly, an error such as a nework problem interferred with the schema retrieval. An example follows:
